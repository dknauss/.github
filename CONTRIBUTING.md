# Contributing

Thanks for contributing.

If the target repository includes its own `CONTRIBUTING.md`, `AGENTS.md`, or other workflow documentation, follow that repository-specific guidance first.

Default expectations:

1. Open a focused branch for non-trivial work.
2. Keep changes scoped to one concern at a time.
3. Run the repository's documented tests, linters, or validation steps before opening a pull request.
4. Update docs, changelogs, or examples when behavior changes.
5. Avoid committing secrets, credentials, or generated noise that the repository does not intentionally track.

For security issues, do not open a public issue. Follow `SECURITY.md` instead.
