# .planning/ Template

Copy this directory into your repo root as `.planning/` to set up GSD workflow tracking.

## Files

| File | Purpose |
|------|---------|
| `PROJECT.md` | Project definition — what it is, requirements, constraints, success criteria |
| `STATE.md` | Current position — phase, status, accumulated context for session continuity |
| `config.json` | GSD workflow configuration |

## Optional additions

As the project grows, GSD will create:
- `ROADMAP.md` — phase breakdown and milestone planning
- `REQUIREMENTS.md` — detailed feature requirements
- `MILESTONES.md` — version milestone tracking
- `phases/` — per-phase plans, research, and verification
- `codebase/` — architecture mapping artifacts
- `research/` — domain research and ecosystem surveys
- `todos/` — task tracking
